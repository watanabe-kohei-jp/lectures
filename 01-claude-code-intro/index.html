混同しがちなので、最初に

「Claude」と「Claude Code」は、別物

モデル（脳）と、ツール（手足） — 別の階層 の話

Claude

"脳" — AI モデル

Anthropic が作る言語モデル本体。賢さの源。

単体では何もできない。会話アプリやツールに「載って」初めて動く。

Claude Code

"手足" — ツール

Claude を載せて、PC を操作するためのツール（Anthropic 公式）。

本講義の主役はこちら。脳ではなく、PC を動かす "手足" の話。

※ 同じ "Claude" でも、Web の claude.ai は「会話するだけ」、Claude Code は「PC を操作する」。同じ脳でも、ツール次第で できることが変わる。次のスライドでは、その Claude Code 自体にも種類があることを見る。

CLI

最も汎用。PC 全体を操作 できるのはこの形だけ。

エディタ内

IDE 拡張

VS Code / JetBrains の中で動く。

コーディング作業に最適化。中身は CLI と同じ。

ブラウザ

Web

claude.ai/code でアクセス。

軽い対話・共有向け。インストール不要。

ネイティブ

Desktop

Mac / Windows のアプリ。

GUI ベース。CLI に慣れない人向け。

※ CLI を扱う理由は「PC 全体を操作する力」が一番強いから。エディタ内・ブラウザ内に閉じない汎用 PC 操作の入口。

Chapter 4 · 立ち位置の変化 06 / 12
この 4 年の結果として — 人間の役割が、初めて変わる

「PC を操作する人」から「指示する人」へ

書く・調べる・整理する — PC 上のすべての行為で、立ち位置が動く

これまで（〜2024）

人間が PC を操作する。AI は会話相手。

キーボード・マウス・ファイル管理は、すべて 人間の手元。

主役の道具は エディタ・Excel・ブラウザ。

これから（2024〜）

人間は 意図を伝える。AI が PC を動かす。

例えば — 情報収集、パワポ作成、Web アプリ操作 まで AI が「勝手に」やる。

主役の道具は ターミナル（AI のコックピット）。

※ 自分で操作できる人の価値が消えるのではなく、AI に上手に指示できる人の価値が新しく生まれた、という話。

理屈はわかった、では具体的に何が頼めるか

PC に頼めることの、ほぼ全領域

日常の作業を 8 つの目的 に切ると、Claude Code はそのほぼ全部に効く

01 · Research

リサーチ

  • 主要論文を 30 本読んで、争点をまとめて
  • 志望企業 20 社の事業と決算を要約して
  • 業界トレンドを直近 1 年で整理して
02 · Code & Data

コーディング・データ

  • Issue #42 を読んで PR まで作って
  • 1 万行の CSV から外れ値と相関を抽出して
  • A/B テストの結果に統計検定をかけて
03 · Automation

自動化

  • 1 週間分の議事録を 1 ページの一覧にまとめて
  • フォルダの PDF を命名ルールでリネームして
  • 形式変換を一括処理化して
04 · Writing

文書作成

  • 議事録 5 本から 1 ページ要約を作って
  • 卒論の章立てと序章ドラフトを書いて
  • 契約書をレビューして改稿案を出して
05 · Slides & Charts

スライド・図表

  • 50 ページの資料を 10 枚スライドに圧縮して
  • データから棒グラフと注釈を作って
  • HTML スライドを動的に生成して
06 · Creative (他 AI 連携)

画像・動画

  • OpenAI Image でヒーロー画像を生成して
  • SNS 用バナーを画像生成 AI で 5 案出して
  • Sora で短い動画クリップを生成して
07 · Communication

連絡管理

  • Slack の長スレ要約と要返信リスト化
  • 100 通のメールから ToDo を抽出して
  • 日本語 → 英語のメール下書きを作って
08 · Learning

勉強・研究

  • ページ最適化の基礎をわかりやすく教えて
  • この目的に対するベストプラクティスを示して
  • 講義ノートを試験対策プリントに整形して

一つの道具で、ここまで覆える。領域の境界が、なくなりつつある。
