課題

AI 周辺は半年〜1 年で道具・ノウハウが入れ替わる。配って終わる教材ではこの速度に追いつけない。

情報は散在し、学ぶ人は 「どれが今も正しいのか」 を判断できない。

必要なもの

AI と人間が共同編集し続ける運用と、それを無料で支える自走する仕組み。

広告枠の設計

編集の中立性を曲げない。教材の主題（Claude Code など）に関連するものに限る。閲覧者から隠さず表示する。

YouTube 的な分配

得られた利益は運営と貢献者で分配する。動機が「評判」だけでなく「金銭」も加わり、ループの回転が太くなる。

売る／売らない · 価値の境界 05 / 09
否定形で価値観を宣言する

何を売り、何を売らないか

売っているもの

リーチ＋正典としての信頼。

教材が信頼される地図であるほど、そこに集う学習者と、その注目の価値が上がる。AI 経由のリーチは AI クローラのログで計測できる。

売らないもの
  • 教材の編集判断
  • 並び順（課金で買えない）
  • 読者の個人データ
  • 媒体に埋め込んだ隠れた指示

※ 媒体は「指令」ではなく「コンテンツ」。AI に伝わる内容は人間にも見える — co-lect の不変条件。

貢献者への還元 · いまから積み上がる 08 / 09
教える側と教わる側の境界はゆるい

4 つの還元

  • 評判 — 消えない、検証できる証跡。 PR で資料を直した人の名前は、Credits・git log・CONTRIBUTORS.md に永続的に残る。typo 修正でも残る。
  • 金銭 — 利益の分配（YouTube 的）。 広告・スポンサー枠から得られる収益の一部を、運営とともに貢献者で分配する。動機に「評判」だけでなく「金銭」も加わる。分配の基準は観客が出てから貢献者と公開の場で決める。
  • 持ち運べる証跡。 GitHub の履歴・Role・Credits は、他の場（就職・受託・登壇）で示せるリンク 1 本の証跡。貢献がそのままポートフォリオになる。
  • 将来の機会への優先。 求人・案件の経路が動き出したとき、貢献の蓄積がある人を優先的に位置づける。早く関わった人ほど証跡を長く積める。
