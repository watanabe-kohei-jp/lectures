# Plan: ドラクエ + Google Drive アナロジーの追加

## Context

既に Claude Design で生成済みの 35 枚スライドデッキ「AI ネイティブ世代の Git/GitHub 入門 #1」（tabId 2010795230）に、**2 つのアナロジー**を組み込む：

### ① ドラクエのセーブデータアナロジー
1. **概念理解の加速** — Git/GitHub と各コマンド（commit/push/pull/conflict）が「ぼうけんのしょ」と直感的に紐づく
2. **conflict の感情的インパクト強化** — 「ぼうけんのしょが消えました！」は世代を超えた強烈なメタファー
3. **次回ブランチへの自然な伏線** — 「ぼうけんのしょ 1, 2, 3」= ブランチの並行作業

### ② Word vs Google Drive アナロジー
1. **Git と GitHub の役割分担を1秒で理解させる** — Word（手元のソフト）/ Google Drive（クラウドで共有）の対応がシンプル
2. **ドラクエを知らない人のセーフティネット** — ドラクエが刺さらなくても、Word/Drive はほぼ全員分かる

聴衆は大学生 5 名。ドラクエを知らない人もいる可能性があるので、ドラクエスライドには軽い一行補足を入れる方針。Word/Google Drive は補足不要。

最終枚数: 35 → **39 枚**

## 各スライドの詳細

### 1. 新規 Slide 11.5「Git をドラクエで例えると」

- **位置**: 概念パートの章扉（Slide 11）直後、commit/push/pull の説明前
- **レイアウト**: comparison（2列対応表）
- **内容**:

| ドラクエ | Git / GitHub |
|---|---|
| ぼうけんのしょ | リポジトリ |
| セーブする | `git commit` |
| メモリーカード（クラウドセーブ） | リモートリポジトリ（GitHub） |
| カードに書き出す | `git push` |
| カードから読み込む | `git pull` |
| ぼうけんのしょ 1, 2, 3 | ブランチ（**次回**） |
| 「ぼうけんのしょが消えました！」 | コンフリクト・データ消失 |

- **左下小さく**: ※ドラクエ＝有名な RPG。セーブ枠が 3 つあるのが伝統。

### 2. 新規 Slide 17.5「もうひとつの例え — Word と Google Drive」

- **位置**: Slide 17「Git と GitHub の違い」（比較表）の直後
- **レイアウト**: analogy（アイコン2つ並べて対比）または content
- **内容**:
  - 上部メッセージ: **「もうひとつの例え方」**
  - 中央: 2 つのアイコンを左右に大きく並べる
    - 左: Word アイコン → 「Git」（手元のソフト、自分のPC内で編集）
    - 右: Google Drive アイコン → 「GitHub」（クラウドで保管・みんなで共有）
  - 下部の一文: 「Word で書いた文書を、Google Drive にアップしてみんなで見る」= 「Git で commit したコードを、GitHub に push してみんなで見る」
- **狙い**:
  - Slide 17 の表の中の 1 行で済ませず、専用ページで視覚的に印象に残す
  - ドラクエが刺さらなかった人のセーフティネット
- **補足は不要**（Word も Google Drive も全員知っている前提でOK）

### 3. 修正 Slide 19「commit の概念図」

- **現状**: `commit = ローカルでセーブする`
- **修正**: タイトル「commit = ぼうけんのしょにセーブ」に変更、図にセーブ画面風の v1→v2→v3 スタックを追加
- **左下小さく**: ※「ぼうけんのしょ」= ドラクエのセーブデータ

### 4. 修正 Slide 22「conflict の概念図」

- **現状**: `conflict = 2人が同じ行を変更してぶつかる`
- **修正**: 副題を「2人が同じ "ぼうけんのしょ" の同じ枠に上書きしようとしたら？」に。次の Slide 22.5 への布石

### 5. 新規 Slide 22.5「ぼうけんのしょが消えました！」

- **位置**: Slide 22 の直後（conflict の感情的補強）
- **レイアウト**: content（メッセージ重視）
- **内容**:
  - 巨大なタイトル: **「ぼうけんのしょが消えました！」**
  - サブメッセージ: 同じ枠に違うデータを上書きしたら、片方は失われる
  - 結論: だから「こまめに commit + push」と「ブランチで枠を分ける」が必要
- **左下小さく**: ※ドラクエ伝統のセーブデータ消失イベントのオマージュ

### 6. 新規 Slide 31.5「ぼうけんのしょ 1, 2, 3 = ブランチ」

- **位置**: Slide 31「だから次回ブランチを学ぶ」の直後（伏線強化）
- **レイアウト**: content + diagram
- **内容**:
  - 上部メッセージ: もし「ぼうけんのしょ」が 1 枠しかなかったら？ → 別シナリオを試せない
  - 中央: ぼうけんのしょ 3 つ並んだ図 → `main` / `feature` / `experiment` ブランチに対応
  - 下部: 「Git のブランチ = ぼうけんのしょを複数持てる仕組み — 次回詳しく」
- **左下小さく**: ※ドラクエはセーブ枠が 3 つあるので、3 つの並行プレイができる
